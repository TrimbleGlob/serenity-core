# Serenity BDD 5.3.6 Release Notes

**Bill of Materials (BOM) for Simplified Dependency Management**

This release introduces the `serenity-bom` module, a Maven Bill of Materials POM that allows consumers to align all Serenity BDD module versions with a single import.

## New Features

### Serenity BDD BOM (`serenity-bom`)

Managing dependencies across a multi-module framework like Serenity BDD has historically required specifying the same version on every artifact. With 40+ modules, this was error-prone and made upgrades tedious.

The new `serenity-bom` artifact solves this by providing a centralized BOM that can be imported in your project's `<dependencyManagement>` section:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>net.serenity-bdd</groupId>
            <artifactId>serenity-bom</artifactId>
            <version>5.3.6</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

Once imported, all Serenity dependencies can be declared without version tags:

```xml
<dependencies>
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-core</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-junit5</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-screenplay-webdriver</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-ensure</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>
```

**Benefits:**
- **Single version to manage** — update one version number to upgrade all Serenity modules
- **No version mismatches** — all modules are guaranteed to be from the same release
- **Cleaner POMs** — less boilerplate in consumer projects
- **Follows ecosystem conventions** — consistent with Spring Boot, JUnit 5, Jackson, and other major Java projects

The BOM covers all published Serenity modules including core, screenplay, test runners, assertions, REST, reporting, cloud provider integrations, and tools.

## Affected Components

| Component | Change |
|-----------|--------|
| `serenity-bom` (new) | New BOM module listing all published Serenity artifacts |
| `pom.xml` (root) | Added `serenity-bom` to all module lists and build profiles |

## Upgrade

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>net.serenity-bdd</groupId>
            <artifactId>serenity-bom</artifactId>
            <version>5.3.6</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

No code changes are required. This is a drop-in replacement for 5.3.5. Existing projects that specify versions explicitly will continue to work — the BOM is opt-in.

# Serenity BDD 5.3.7 Release Notes

**New BOM for Simplified Dependency Management, CI Fixes, and Playwright Test Stabilization**

This release introduces the `serenity-bom` module for easy version alignment across all Serenity artifacts, fixes the release workflow, upgrades all CI actions to Node.js 24, and replaces flaky external test dependencies with local fixtures.

## New Features

### Serenity BDD BOM (`serenity-bom`)

Managing dependencies across Serenity BDD's 40+ modules has historically required specifying the same version on every artifact. The new `serenity-bom` artifact provides a centralized Bill of Materials that you can import in your project's `<dependencyManagement>` section:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>net.serenity-bdd</groupId>
            <artifactId>serenity-bom</artifactId>
            <version>5.3.7</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

Once imported, declare Serenity dependencies without version tags:

```xml
<dependencies>
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-core</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-junit5</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-screenplay-webdriver</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>net.serenity-bdd</groupId>
        <artifactId>serenity-ensure</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>
```

The BOM covers all published Serenity modules including core, screenplay, test runners, assertions, REST, reporting, cloud provider integrations, and tools. When upgrading, you only need to change the single version number in the BOM import.

See the [Maven documentation](https://serenity-bdd.github.io/docs/guide/maven) for complete examples with JUnit 5 and Cucumber.

## Bug Fixes

### Release Workflow Failure

The release workflow was failing with two errors:
- **Missing `distributionManagement`** in `serenity-bom` — the BOM module didn't inherit the parent POM's publishing configuration. Fixed by adding the parent POM reference.
- **Wrong release profile name** — the maven-release-plugin referenced a profile named `release` that didn't exist. Fixed to reference the actual `deploy` profile.

### Playwright iframe Tests Failing

The `WithinFrameInteractionsTest` tests were failing because the external test site (`the-internet.herokuapp.com/iframe`) changed its TinyMCE editor to load in read-only mode (`contenteditable="false"`). Replaced the external dependency with local HTML fixtures to eliminate flaky test behavior.

## Infrastructure

### GitHub Actions Upgraded to Node.js 24

All CI workflows have been upgraded to use the latest GitHub Action versions that support Node.js 24, ahead of the June 2026 deprecation deadline:

| Action | Previous | Updated |
|--------|----------|---------|
| `actions/checkout` | v3/v4 | **v6** |
| `actions/setup-java` | v3/v4 | **v5** |
| `actions/cache` | v3/v4 | **v5** |
| `GabrielBB/xvfb-action` | v1 | Replaced with `xvfb-run` |

The PR build workflow was also fixed to use JDK 17 (was incorrectly set to JDK 11).

## Affected Components

| Component | Change |
|-----------|--------|

| `serenity-bom/pom.xml` | Inherits from parent POM for correct publishing behavior |
| `pom.xml` (root) | Fixed `releaseProfiles` from `release` to `deploy` |

## Upgrade

Using the BOM (recommended):

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>net.serenity-bdd</groupId>
            <artifactId>serenity-bom</artifactId>
            <version>5.3.7</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

Or update the version directly:

```xml
<dependency>
    <groupId>net.serenity-bdd</groupId>
    <artifactId>serenity-core</artifactId>
    <version>5.3.7</version>
</dependency>
```

No code changes are required. This is a drop-in replacement for 5.3.6.
